# x402 Payments

Pay for elizaOS Cloud services with cryptocurrency using the x402 payment protocol.

<div className="status-badge status-stable">Stable</div>

## Overview

x402 enables:

- **Crypto Payments**: Pay with USDC on supported networks
- **Instant Settlement**: No waiting for confirmations
- **API-First**: Programmatic payment integration
- **Self-Custody**: You control your funds

## Supported Networks

| Network  | Token | Address                                      |
| -------- | ----- | -------------------------------------------- |
| Base     | USDC  | `0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913` |
| Ethereum | USDC  | `0xA0b86991c6218b36c1d19D4a2e9Eb0cE3606eB48` |

## How It Works

1. **Request**: Make an API call without payment
2. **402 Response**: Server returns payment requirements
3. **Pay**: Submit payment proof in `X-PAYMENT` header
4. **Fulfill**: Request completes with valid payment

## Payment Flow

### 1. Initial Request (No Payment)

```bash
curl -X POST "https://elizacloud.ai/api/v1/chat/completions" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-4o",
    "messages": [{"role": "user", "content": "Hello"}]
  }'
```

### 2. Receive 402 Response

```json
{
  "error": {
    "code": "PAYMENT_REQUIRED",
    "message": "Payment required for this request"
  }
}
```

Headers:

```
X-Payment-Requirement: {
  "accepts": [{
    "scheme": "exact",
    "network": "base",
    "maxAmountRequired": "10000",
    "resource": "https://elizacloud.ai/api/v1/chat/completions",
    "payTo": "0x...",
    "extra": {
      "name": "elizaOS Cloud",
      "description": "Chat completion"
    }
  }]
}
```

### 3. Make Payment

Create and sign a payment transaction, then include proof:

```bash
curl -X POST "https://elizacloud.ai/api/v1/chat/completions" \
  -H "Content-Type: application/json" \
  -H "X-PAYMENT: <payment-header>" \
  -d '{
    "model": "gpt-4o",
    "messages": [{"role": "user", "content": "Hello"}]
  }'
```

## JavaScript Integration

### Using @anthropic-ai/x402

```javascript
import { wrapFetch } from "@anthropic-ai/x402";
import { createWalletClient, http } from "viem";
import { base } from "viem/chains";
import { privateKeyToAccount } from "viem/accounts";

// Set up wallet
const account = privateKeyToAccount("0x...");
const wallet = createWalletClient({
  account,
  chain: base,
  transport: http(),
});

// Wrap fetch with x402
const x402Fetch = wrapFetch(fetch, wallet);

// Make request - payment handled automatically
const response = await x402Fetch(
  "https://elizacloud.ai/api/v1/chat/completions",
  {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({
      model: "gpt-4o",
      messages: [{ role: "user", content: "Hello!" }],
    }),
  },
);
```

## Payment Header Format

The `X-PAYMENT` header contains a signed payment proof:

```
X-PAYMENT: x402;scheme=exact;network=base;amount=10000;signature=0x...;txHash=0x...
```

| Field       | Description                 |
| ----------- | --------------------------- |
| `scheme`    | Payment scheme (exact)      |
| `network`   | Blockchain network          |
| `amount`    | Payment amount in wei       |
| `signature` | Payment signature           |
| `txHash`    | Transaction hash (optional) |

## Security

<Callout type="warning">
  **Private Key Security**: Never expose your private keys in client-side code.
  Use a secure backend to sign payments.
</Callout>

### Best Practices

- Use hardware wallets for production
- Set spending limits
- Monitor transaction history
- Use test networks for development

## Error Handling

| Error                  | Cause                 | Solution            |
| ---------------------- | --------------------- | ------------------- |
| `INVALID_PAYMENT`      | Invalid payment proof | Check signature     |
| `INSUFFICIENT_PAYMENT` | Amount too low        | Increase payment    |
| `EXPIRED_PAYMENT`      | Payment expired       | Generate new proof  |
| `WRONG_NETWORK`        | Network mismatch      | Use correct network |

## Best Practices

- **Secure Keys** — Never expose private keys in client-side code; use a secure backend
- **Monitor Balance** — Track wallet balance to ensure sufficient funds for payments
- **Test First** — Use test networks during development before mainnet
- **Set Limits** — Configure spending limits to prevent unexpected charges
